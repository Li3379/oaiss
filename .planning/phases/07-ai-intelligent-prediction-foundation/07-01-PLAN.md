    - Three methods, each with retry logic:
      1. `public MarketForecastResponse predictMarket(MarketForecastRequest request)` -- POST to `/api/v1/predict/market`, bodyValue(request), retrieve().bodyToMono(MarketForecastResponse.class), retryWhen(backoff with filter on WebClientException), block()

    - Retry: `Retry.backoff(retryMaxAttempts, Duration.ofMillis(retryBackoffMillis)).filter(ex -> ex instanceof WebClientException)`
    - On 422: do NOT retry. Add `.onStatus(status -> status.value() == 422, response -> ...)` that throws BusinessException immediately

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Spring Boot -> ML Service | WebClient HTTP calls cross network boundary; ML service may be down or return invalid data |
| Client -> Spring Boot | User input reaches prediction endpoints; must validate before sending to ML service |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | MlServiceClient | mitigate | Pydantic validates all request fields with Field constraints (ge, le, min_length); Java DTOs use jakarta.validation |
| T-07-02 | Denial of Service | ML Service endpoints | mitigate | WebClient retry with backoff (3 attempts, 1s/2s/4s); circuit breaker pattern on repeated failures |
| T-07-03 | Information Disclosure | MlServiceClient | mitigate | ML service has no direct DB access; Java side sends only aggregated feature vectors, not raw PII |
| T-07-04 | Tampering | Python dependencies | mitigate | Pin exact versions in requirements.txt (fastapi==0.115.0, etc.); use Docker for reproducible builds |
</threat_model>
