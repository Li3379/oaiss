    **oaiss-chain-ml/app/services/market_service.py** -- `MarketService` class:

    1. `__init__(self)` -- Set `self.model_version = "1.0.0"`, `self._prophet_model = None`, `self._xgb_model = None`

    2. `predict(self, request: MarketForecastRequest) -> MarketForecastResponse`:
       - Validate input: `len(request.dates) == len(request.prices) == len(request.volumes)`, raise ValueError if not
       - Parse `request.dates` strings to `pd.Timestamp` using `pd.to_datetime()`
       - Build Prophet DataFrame: `df = pd.DataFrame({"ds": dates, "y": prices})`
       - Fit Prophet model: `model = Prophet(interval_width=0.8, changepoint_prior_scale=0.05)`; `model.fit(df)`
       - Create future dataframe: `future = model.make_future_dataframe(periods=request.horizon_days)`
       - Forecast: `forecast = model.predict(future)`; extract last `horizon_days` rows
       - Extract `yhat` (forecast_prices), `yhat_lower` (lower_bound), `yhat_upper` (upper_bound)
       - Train XGBoost for supply/demand as secondary signal (volume trend):
         - Features: day-of-week, day-of-month, month, lag-7 volume, lag-30 volume (computed from input volumes)
         - Target: next-day volume
         - `xgb.XGBRegressor(n_estimators=100, max_depth=3, learning_rate=0.1, objective='reg:squarederror')`
         - Predict next `horizon_days` volumes using iterative prediction
       - Determine trend: compute `forecast_prices[-1] - forecast_prices[0]`; if delta > 2% of first price -> "up", if < -2% -> "down", else "stable"
       - Build response with `forecast_dates` as ISO 8601 strings, `forecast_prices`, `lower_bound`, `upper_bound`, `trend`, `model_version`
       - Log prediction duration and trend at INFO level
       - Return `MarketForecastResponse`

    3. Error handling: wrap Prophet/XGBoost calls in try/except; if Prophet fails (e.g., insufficient data points < 2), raise `HTTPException(status_code=422, detail="Insufficient data for forecasting: need at least 2 data points")`
