<objective>
Security and code quality improvements: encrypt RSA private keys at rest (SEC-01), document CSRF disable decision (SEC-02), add readOnly=true to genuinely read-only @Transactional methods (QUAL-01), and add AndDeletedFalse to soft-delete queries (QUAL-02).

Purpose: Private keys stored as plaintext Base64 in MySQL are a key compromise risk if the database is breached. Missing readOnly hints force Hibernate to track dirty checking unnecessarily. Soft-delete queries without AndDeletedFalse return logically deleted records to business logic.
Output: AesGcmEncryptor utility, updated DigitalSignatureService, V7 Flyway migration, ADR-001 document, ~12 readOnly=true annotations, 11 AndDeletedFalse method renames across 6 repositories.

    Structure:
    - **Title**: ADR-001: Disable CSRF Protection
    - **Status**: Accepted
    - **Date**: 2026-05-20
    - **Context**: Spring Security 6.x enables CSRF by default with deferred tokens. Our API is stateless JWT-based. Tokens stored in sessionStorage (not cookies). CSRF attacks exploit automatic cookie submission -- not applicable when browsers do not send auth tokens automatically.
    - **Decision**: Disable CSRF via `.csrf(AbstractHttpConfigurer::disable)` in SecurityConfig. Spring Security 6.x deferred CSRF tokens caused POST requests to return 401 (misinterpreted as session expiry) instead of 403.
    - **Alternatives Considered**:
      1. Enable CSRF with CookieCsrfTokenRepository -- adds complexity, no security benefit for sessionStorage JWT
      2. Enable CSRF with custom header -- requires frontend changes, no benefit over JWT-in-header approach
    - **Consequences**: Stateless API remains fully RESTful. No session state. Client must send JWT in Authorization header for every request. If future changes move to cookie-based auth, this decision must be revisited.
    - **References**: SecurityConfig.java lines 54-58, OWASP CSRF Prevention Cheat Sheet

    This is documentation-only -- no code changes. The inline comment in SecurityConfig already explains the rationale; the ADR formalizes it.
