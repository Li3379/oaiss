    **BUG-01 Fix (DigitalSignatureService.java):**
    Replace ALL occurrences of `findByUserIdAndDeletedFalse(userId)` with `findLatestByUserId(userId)` in DigitalSignatureService.java. There are 6 call sites: lines 120, 137, 161, 202, 247, 284 (approximate). The method `findLatestByUserId` already exists in RsaKeyPairRepository and uses `ORDER BY r.keyVersion DESC LIMIT 1` which prevents NonUniqueResultException. This is per RESEARCH.md BUG-01 fix and CONTEXT.md D-15.

    **BUG-02 Fix (SecurityConfig.java) -- per D-06:**
    Change the Swagger endpoint security from `.permitAll()` to `.authenticated()`. Replace lines 72-77:
    ```java
    // BEFORE:
    .requestMatchers(
        "/swagger-ui/**",
        "/swagger-ui.html",
        "/v1/api-docs/**",
        "/v3/api-docs/**"
    ).permitAll()

    // AFTER:
    .requestMatchers(
        "/swagger-ui/**",
        "/swagger-ui.html",
        "/v1/api-docs/**",
        "/v3/api-docs/**"
    ).authenticated()
    ```
    This is safe because application-docker.yml already disables springdoc entirely via `springdoc.swagger-ui.enabled: false`, so the authenticated() rule only applies in dev/default profile.

    **BUG-03 Fix (SecurityConfig.java + application.yml) -- per D-07:**
    In SecurityConfig.java line 44, change:
    ```java
    // BEFORE:
    @Value("${app.cors.allowed-origins:http://localhost:5173}")
    private List<String> allowedOrigins;

    // AFTER:
    @Value("${app.cors.allowed-origins}")
    private List<String> allowedOrigins;
    ```
    In application.yml line 134, the value already reads `${CORS_ALLOWED_ORIGINS:http://localhost:5173}` -- this is the YAML-level default, which is correct for dev mode. The @Value annotation was adding a SECOND default layer. Removing the @Value default means the YAML default still provides the dev fallback. This is the simplest fix per RESEARCH.md recommendation.

    **After all fixes, recompile and restart the backend:**
    ```bash
    cd oaiss-chain-backend && mvn compile -q && mvn spring-boot:run -Dspring-boot.run.jvmArguments="-Dfile.encoding=UTF-8" &

    ```
