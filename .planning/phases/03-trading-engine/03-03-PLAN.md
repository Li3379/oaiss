From TradeController.java (backend):
- POST /api/v1/trade/p2p -- ENTERPRISE role. Body: {"tradeType":2, "sellerId":long, "buyerId":long, "quantity":BigDecimal, "unitPrice":BigDecimal, "reportId":null, "remark":"string"}
  - Validates: tradeType==2, sellerId!=buyerId, currentUser.userId==sellerId, enterprise.carbonTradable>=quantity
  - Returns {code:200, data:{id, tradeType, sellerId, buyerId, quantity, unitPrice, status, ...}}
  - status after creation: PENDING (0)
- POST /api/v1/trade/{tradeId}/confirm -- ENTERPRISE role.
  - Only participants (sellerId or buyerId == currentUserId) can confirm
  - Status transitions: PENDING -> PROCESSING -> COMPLETED (atomic)
  - Settlement: seller loses carbonTradable, gains carbonUsed; buyer gains carbonQuota + carbonTradable
  - Returns {code:200, data:{...status:2 (COMPLETED)}}
- POST /api/v1/trade/{tradeId}/cancel -- ENTERPRISE role.
  - Only cancellable statuses (PENDING, PROCESSING)
  - Transitions to CANCELLED (3)
  - Returns {code:200, data:{...status:3 (CANCELLED)}}
