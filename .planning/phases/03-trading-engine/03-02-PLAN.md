From DoubleAuctionController.java (backend):
- POST /api/v1/auction/buy -- ENTERPRISE role. Body: {"quantity":BigDecimal, "price":BigDecimal}
  - Validates enterprise.carbonTradable >= quantity
  - Returns {code:200, data:{id, orderNo, userId, quantity, price, status, orderType, ...}}
  - Error if insufficient quota: throws TradeException
- POST /api/v1/auction/sell -- ENTERPRISE role. Body: {"quantity":BigDecimal, "price":BigDecimal}
  - Same validation as buy: enterprise.carbonTradable >= quantity
  - Returns same structure as buy
- POST /api/v1/auction/match -- ADMIN role. No body.
  - Triggers DoubleAuctionService.executeMatching() (synchronized method)
  - Returns list of MatchingResult
  - Matching rules: buy orders price DESC, sell orders price ASC, match when buyPrice >= sellPrice
  - Settlement price = (buyPrice + sellPrice) / 2
  - Matched quantity = min(buyRemaining, sellRemaining)
- GET /api/v1/auction/my-orders -- ENTERPRISE role. Returns list of user's orders with status.
- GET /api/v1/auction/results?page=1&size=10 -- Returns matching results.

AuctionOrderStatusEnum: PENDING(0), PARTIALLY_MATCHED(1), FULLY_MATCHED(2), CANCELLED(3)

Settlement quota update logic (DoubleAuctionService lines 338-352):
  seller.setCarbonTradable(seller.getCarbonTradable() - quantity)
  seller.setCarbonUsed(seller.getCarbonUsed() + quantity)
  buyer.setCarbonQuota(buyer.getCarbonQuota() + quantity)
  buyer.setCarbonTradable(buyer.getCarbonTradable() + quantity)
