    `@Transactional public EnterpriseAdmission issueCertificate(Long enterpriseId)`:
    1. Verify enterprise exists: `enterpriseRepository.findById(enterpriseId).orElseThrow(() -> BusinessException.notFound("error.enterprise.notFound"))`
    2. Check duplicate ACTIVE per D-07: `enterpriseAdmissionRepository.findByEnterpriseIdAndStatusAndDeletedFalse(enterpriseId, 1)` -- if `isPresent()`, throw `BusinessException.of(ErrorCode.PARAM_ERROR, "error.admission.alreadyActive")`

    4. Build and save: `EnterpriseAdmission.builder().enterpriseId(enterpriseId).certificateNo(certificateNo).issuedDate(LocalDate.now()).status(1).build()` then `enterpriseAdmissionRepository.save(admission)`
    5. Log: `log.info("Enterprise admission issued: {} for enterprise {}", certificateNo, enterpriseId)`
    6. Return saved entity

    `@Transactional public void revokeCertificate(Long enterpriseId)`:
    1. Find ACTIVE: `enterpriseAdmissionRepository.findByEnterpriseIdAndStatusAndDeletedFalse(enterpriseId, 1).stream().findFirst().orElseThrow(() -> BusinessException.notFound("error.admission.notFound"))`
    2. Set status = 2 (REVOKED) per D-08
    3. Save and log

    3. Else: return `findByDeletedFalse(pageable)`

    1. Return `enterpriseAdmissionRepository.findByEnterpriseIdAndDeletedFalse(enterpriseId)`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client->API | Untrusted HTTP requests cross into admin endpoints |
| ENTERPRISE -> /my | Non-admin users access /my endpoint with their own JWT |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | POST /enterprise-admission/{id}/issue | mitigate | @PreAuthorize("hasRole('ADMIN')") at class level ensures only admins can issue certificates. enterpriseId validated against DB. |
| T-10-02 | Tampering | DELETE /enterprise-admission/{id} | mitigate | Same class-level @PreAuthorize. Only ACTIVE certificates can be revoked (status check in service). |
| T-10-03 | Information Disclosure | GET /enterprise-admission (list) | mitigate | ADMIN-only endpoint. Returns paginated data, no sensitive fields beyond cert metadata. |
| T-10-04 | Elevation of Privilege | GET /enterprise-admission/my | mitigate | Method-level @PreAuthorize("hasRole('ENTERPRISE')") overrides class-level. JWT userId -> enterpriseId lookup ensures user can only see own certs. |
| T-10-05 | Spoofing | Certificate number generation | accept | EA- prefix + date + 6-digit random. Not cryptographically secure but sufficient for internal cert IDs. Collision risk negligible at expected scale. |
| T-10-06 | Denial of Service | Duplicate issuance spam | mitigate | D-07: ACTIVE check prevents duplicate certificates per enterprise. |
</threat_model>
