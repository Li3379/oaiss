## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | POST /admin/enterprise-admission/{id}/issue | mitigate | @PreAuthorize("hasRole('ADMIN')") at class level; enterpriseId validated against DB in service |
| T-10-02 | Tampering | DELETE /admin/enterprise-admission/{id} | mitigate | Same class-level ADMIN guard; only ACTIVE certificates can be revoked (status check in service) |
| T-10-03 | Information Disclosure | GET /admin/enterprise-admission (list) | mitigate | ADMIN-only; no sensitive data beyond cert metadata |
| T-10-04 | Elevation of Privilege | GET /admin/enterprise-admission/my | mitigate | Method-level @PreAuthorize("hasRole('ENTERPRISE')") overrides class-level ADMIN; enterpriseId from JWT claims (currentUser.getEnterpriseId()), not user-supplied parameter |
| T-10-05 | Spoofing | Certificate number generation | accept | EA- prefix + date + 6-digit random is not cryptographically secure but sufficient for internal cert IDs; collision handled by retry loop + DB UNIQUE constraint |
| T-10-06 | Denial of Service | Duplicate issuance spam | mitigate | D-07: ACTIVE check prevents duplicate certificates per enterprise; DB UNIQUE on certificate_no prevents collision |
</threat_model>
