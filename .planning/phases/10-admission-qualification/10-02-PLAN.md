<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client->API | Untrusted HTTP requests cross into admin endpoints |
| REVIEWER -> /my | Non-admin users access /my endpoint with their own JWT |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-08 | Tampering | POST /reviewer-qualification/{id}/issue | mitigate | @PreAuthorize("hasRole('ADMIN')") at class level ensures only admins can issue qualifications. reviewerId validated against DB. |
| T-10-09 | Tampering | DELETE /reviewer-qualification/{id} | mitigate | Same class-level @PreAuthorize. Only ACTIVE qualifications can be revoked (status check in service). |
| T-10-10 | Information Disclosure | GET /reviewer-qualification (list) | mitigate | ADMIN-only endpoint. Returns paginated data, no sensitive fields beyond cert metadata. |
| T-10-11 | Elevation of Privilege | GET /reviewer-qualification/my | mitigate | Method-level @PreAuthorize("hasRole('REVIEWER')") overrides class-level. JWT userId -> reviewerId lookup ensures user can only see own certs. |
| T-10-12 | Spoofing | Certificate number generation | accept | RQ- prefix + date + 6-digit random. Not cryptographically secure but sufficient for internal cert IDs. |
| T-10-13 | Denial of Service | Duplicate issuance spam | mitigate | D-07: ACTIVE check prevents duplicate qualifications per reviewer. |
</threat_model>
