  <action>
    Per D-01: Create FabricCAService as @Service with @Profile("fabric") that implements a single registerEnrollment() method for gateway admin enrollment only (not multi-tenant).

    Per D-04: The service reads config from FabricProperties.Ca (injected). It uses Spring WebClient (already in project via spring-boot-starter-webflux for ML service) to call Fabric CA REST API directly at POST /api/v1/enroll with Basic auth (adminName:adminPassword). This avoids the deprecated fabric-sdk-java HFCAClient dependency (per RESEARCH.md recommendation).

    FabricCAService.registerEnrollment() flow:
    1. Generate an EC key pair using BouncyCastle (bcpkix-jdk18on:1.78.1, already in pom.xml)
    2. Build a PKCS10 CSR with the key pair and adminName as CN
    3. POST the CSR to FabricProperties.Ca.endpoint + "/api/v1/enroll" with Basic auth header
    4. Parse the JSON response: extract base64-encoded "cert" (enrollment certificate) and "ServerInfo.CAChain" (CA certificate chain)
    5. Decode the enrollment cert into X509Certificate via Identities.readX509Certificate
    6. Construct and return an EnrollmentResult record containing: Identity (new X509Identity(mspId, cert)) and Signer (Signers.newPrivateKeySigner(privateKey))

    EnrollmentResult is a package-private record -- not a JPA entity, not a DTO, not serializable. It is consumed only by FabricGatewayConfig.

    Per T-18-01: Add @ToString.Exclude to FabricProperties.Ca.adminPassword field. The Ca inner class currently uses @Data from Lombok which generates toString() including adminPassword -- this must be excluded.

    Modify application-fabric.yml: Add fabric.ca configuration section under the existing `fabric:` block:
    ```yaml
    fabric:
      enabled: true
      ca:
        enabled: false
        endpoint: http://ca.org1.example.com:7054
        admin-name: admin
        admin-password: ${FABRIC_CA_ADMIN_PASSWORD:}
    ```

    Do NOT add any new Maven dependencies -- spring-boot-starter-webflux and bcpkix-jdk18on already exist in pom.xml. Do NOT add fabric-ca-sdk (deprecated, per RESEARCH.md).

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Information Disclosure | FabricProperties.Ca.toString() | mitigate | @ToString.Exclude on adminPassword field prevents Lombok from including it in toString() output, which blocks accidental logging |
| T-18-02 | Information Disclosure | EnrollmentResult serialization | mitigate | EnrollmentResult is a package-private record with no Jackson annotations, no JPA mapping, no DTO exposure -- never serialized or persisted |
| T-18-03 | Spoofing | CA enrollment response | mitigate | WebClient call uses the configured CA endpoint from FabricProperties.Ca.endpoint; TLS can be enforced via https:// scheme in config |
| T-18-04 | Tampering | CA REST API response | accept | CA is a trusted infrastructure component within the Fabric network; response integrity is the CA's responsibility |
| T-18-SC | Tampering | Maven/npm installs | mitigate | All dependencies already exist in pom.xml (spring-boot-starter-webflux, bcpkix-jdk18on) -- no new package installs required |
</threat_model>
