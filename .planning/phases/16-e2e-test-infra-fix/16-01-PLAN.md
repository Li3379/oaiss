# Plan 16-01: Auth Fixture Fix

**Phase:** 16-e2e-test-infra-fix
**Requirement:** E2E-01
**Depends on:** Phase 15 (shipped)
**Estimate:** 1 task, ~5 min

## Goal

Fix `loginViaApi()` sending form-encoded data instead of JSON, which causes 30s timeout on Spring Boot `@RequestBody` endpoints. This single-line fix unblocks 68+ flow tests and 10+ v1.1 tests.

## Root Cause

`auth.ts:45-47` — Playwright's `page.request.post(url, { data: obj })` defaults to `application/x-www-form-urlencoded`. Spring Boot `AuthController.login()` expects `@RequestBody` JSON.

## Tasks

### Task 1: Add Content-Type header to loginViaApi()

**File:** `oaiss-chain-frontend/tests/e2e/fixtures/auth.ts`

**Change:** At line 45-47, add `headers` to the `page.request.post()` call:

```typescript
// BEFORE (line 45-47):
const response = await page.request.post(`${API_BASE}/auth/login`, {
  data: { username, password },
})

// AFTER:
const response = await page.request.post(`${API_BASE}/auth/login`, {
  headers: { 'Content-Type': 'application/json' },
  data: { username, password },
})
```

**No other changes needed.** Token storage strategy (D-02) stays as-is: `loginViaApi` uses localStorage, `loginViaToken` uses sessionStorage. Frontend `getToken()` checks both.
