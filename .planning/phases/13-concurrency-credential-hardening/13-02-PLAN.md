<objective>
Externalize all hardcoded credentials from infrastructure configuration and remove insecure defaults.

Purpose: Hardcoded passwords in docker-compose.yml and application.yml are the most common attack vector for infrastructure compromise. DB_PASSWORD:123456 and minioadmin:minioadmin are well-known defaults that attackers try first. Removing defaults forces operators to set real credentials.
Output: All YAML files use environment variable substitution with no insecure defaults. .env.example documents every required variable.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Config files -> deployment | Credentials in YAML are visible in source control and container images |
| Developer machine -> production | Default credentials in .env.example must not ship to production |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-04 | Information Disclosure | docker-compose.yml hardcoded passwords | mitigate | Replace all literals with ${VAR} substitution (SEC-07) |
| T-13-05 | Spoofing | DB_PASSWORD:123456 default | mitigate | Remove default, fail-fast on missing env var (SEC-08) |
| T-13-06 | Spoofing | MinIO minioadmin default credentials | mitigate | Remove default, use env vars only (SEC-09) |
| T-13-07 | Tampering | SecurityStartupValidator | mitigate | Reject weak credential values at application startup |
</threat_model>
