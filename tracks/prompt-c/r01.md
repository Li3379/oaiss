# Scheme C — Round 1

**Scheme**: C (安全/生产就绪)
**Round**: 1
**Change**: 安全加固 + 部署编排（与 Prompt-A R1 相同底层修复，C 视角聚焦安全维度）
**Biggest issue from last round**: JWT secret 硬编码 fallback 可被用于伪造 token；Actuator /env 端点暴露数据库密码

## 安全维度修复清单

| 风险 | 修复前 | 修复后 |
|------|--------|--------|
| JWT secret 硬编码 | `${JWT_SECRET:oaiss-chain-jwt-secret-...}` | `${JWT_SECRET}`（无 fallback，启动失败强制配置） |
| CORS 全开放 | `setAllowedOriginPatterns(List.of("*"))` | `setAllowedOrigins(allowedOrigins)` 从配置读取 |
| Actuator 暴露 | `/actuator/**` permitAll | 仅 health/prometheus 公开，其余需认证 |
| SecurityConfig 路径无效 | `/enterprise/**` 等不匹配任何 Controller | 移除，由 `@PreAuthorize` 统一管控 |
| 无 Flyway 迁移 | `classpath:db/migration` 目录为空 | `V1__init_schema.sql` 就位 |
| ddl-auto: update | 生产环境自动 DDL | `validate`（Flyway 接管） |
| 无 Docker 编排 | 手动搭建 5 个服务 | `docker-compose.yml` 一键启动 |
| 无开发环境配置 | 所有环境共用 application.yml | `application-dev.yml` 独立开发配置 |
