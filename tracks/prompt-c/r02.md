# Scheme C — Round 2

**Scheme**: C (安全/生产就绪)
**Round**: 2
**Change**: 修复 AdminController 密码哈希泄露风险 — User 实体 password 字段添加 @JsonProperty(WRITE_ONLY)
**Biggest issue from last round**: AdminController.listUsers() 返回 Page<User>，password 字段（BCrypt hash）会被序列化到 JSON 响应中

## Changes applied

1. `User.java`: password 字段添加 `@JsonProperty(access = JsonProperty.Access.WRITE_ONLY)` — 序列化时永远不输出密码字段

## 安全影响

| 修复前 | 修复后 |
|--------|--------|
| GET /admin/users 返回包含 password 字段的 User JSON | password 字段在序列化时被跳过 |
| BCrypt hash 可被管理员看到 | password 仅在反序列化（注册/修改密码请求）时读取 |
