# E2E Testing: Full-Role, Full-Module Coverage — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a comprehensive E2E testing suite covering all 5 roles and 16 business modules, with Playwright frontend tests + Testcontainers backend integration tests.

**Architecture:** Role-layered approach with dual test modes — Mock-based smoke tests for fast page verification, real API business flow tests for end-to-end validation. Page Object Model for maintainability, per-scenario iterative development loop.

**Tech Stack:** Playwright 1.59 (frontend E2E), Testcontainers 1.19.7 (backend integration), JUnit 5, Spring Boot Test, TypeScript

        ResponseEntity<String> response = restTemplate.postForEntity("/api/v1/auth/login", request, String.class);
        // Extract token from response

            var jsonNode = objectMapper.readTree(response.getBody());
            return jsonNode.path("data").path("accessToken").asText();

        headers.setBearerAuth(token);
